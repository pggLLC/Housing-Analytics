# `js/place-chas-coverage-panel.js`

place-chas-coverage-panel.js

Renders the "Place-Level Data Coverage" panel on
dashboard-data-quality.html. Surfaces transparency metrics for the
TIGER spatial-join place-CHAS pipeline so analysts can see at a
glance which places get tract-aggregated CHAS vs which fall back to
county-level data.

Why this matters
----------------
The dashboard mixes two CHAS methodologies depending on the place:
  1. Tract-aggregated (TIGER 2024 spatial join) — accurate even for
     cross-county jurisdictions like Erie, Aurora, Longmont
  2. Primary-county fallback — used when the place isn't covered by
     the TIGER pipeline (~4% of places as of 2026-05-10)

Without this panel, a user can't tell which methodology a given
place uses. The panel shows the buckets explicitly so the
methodological provenance is auditable.

Data source: data/hna/place-chas-coverage-stats.json
  (regenerated by scripts/hna/build_place_chas_coverage_stats.py
   every time place-chas.json or the alias map updates)
