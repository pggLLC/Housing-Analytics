# `js/cross-county-disclosure.js`

cross-county-disclosure.js

Helper for surfacing a "this jurisdiction crosses county lines" warning
when the user is modeling a CO place that spans multiple counties.

Why this matters
----------------
HUD AMI is set at the county level (HMFA). 26 CO places — including
Aurora, Erie, Boulder, Longmont, Northglenn, Superior, Windsor, etc. —
physically straddle county boundaries. The Deal Calculator and PMA
simulator pick the place's *primary* county (highest population share)
for AMI lookups, but a parcel on the wrong side of the line uses a
different county's HUD AMI. The 2025 HAMFI delta between Boulder and
Weld is ~25%; for a 60% AMI deal, that's a 25% rent delta.

Public API
----------
  window.CrossCountyDisclosure.init() — fetch + cache the data file
  window.CrossCountyDisclosure.lookup(geoid) — returns null or:
    {
      name: "Aurora (city)",
      primary_county: "08005",
      all_counties: [{fips, name, population}, ...]
    }
  window.CrossCountyDisclosure.lookupByCountyFips(countyFips) — returns
    a list of cross-county places where this county is one of the
    touched counties (primary OR secondary). Useful for surfacing
    "If your site is in [X], confirm the parcel's actual county."
  window.CrossCountyDisclosure.formatBanner(place) — returns ready-to-
    render HTML for the disclosure banner.

Data source
-----------
  data/hna/cross-county-places.json  (regenerated by
  scripts/hna/build_cross_county_places.py)

## Symbols

### `lookup(geoid)`

Lookup by 7-digit place geoid. Returns the place record or null.

### `lookupByCountyFips(countyFips)`

Lookup all cross-county places that touch a given county.

### `formatBanner(placeRecord)`

Build banner HTML for a place record. Returns '' for non-cross-county.

### `formatCountyBanner(countyFips)`

Build a county-level summary banner: lists cross-county places in this
 county. Returns '' if none. Useful when only the county is known
 (e.g. Deal Calculator before a site is geocoded).
