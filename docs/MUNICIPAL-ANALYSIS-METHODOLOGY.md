# Municipal Analysis Methodology — Sub-County Scaling for the HNA

> **Module:** `js/municipal-analysis.js`
> **Data files:** `data/hna/municipal/`
> **Version:** 1.0 · **Last updated:** 2024-11-15

---

## Overview

The Housing Needs Assessment (HNA) tool provides municipal-level analysis for Colorado
cities, towns, and Census-Designated Places (CDPs) by scaling county-level ACS and DOLA
data down to the sub-county geography. This document describes the statistical methodology,
data sources, confidence levels, and limitations of the sub-county scaling framework.

---

## 1. Data Architecture

### 1.1 Source Hierarchy

| Priority | Data source | Coverage | Reliability |
|---|---|---|---|
| 1 (highest) | Direct ACS 5-year place-level tables | Places ≥ 2,500 pop | High (direct Census) |
| 2 | ACS 5-year place-level tables | Places < 2,500 pop | Moderate (large MOE) |
| 3 | County ACS × population share | All places | Interpolated estimate |
| 4 | County trend extrapolation | All places | Estimated |
| 5 (lowest) | No data available | — | Unavailable |

### 1.2 File Structure

```
data/hna/municipal/
├── municipal-config.json        — All featured CO municipalities: FIPS, population,
│                                   county, direct ACS metrics
├── growth-rates.json            — 3-yr, 5-yr, 10-yr CAGRs and smoothed projection rate
├── scaling-factors/{fips5}.json — (Generated by ETL) Per-county scaling inputs
├── demographics/{geoid}.json    — (Generated by ETL) Pre-calculated demographic baselines
└── affordability/{geoid}.json   — (Generated by ETL) Affordability tier distributions
```

## 2. Population & Household Scaling

### 2.1 Municipal Population Share

```
pop_share = municipal_population / county_population
```

Where both values come from ACS 5-year DP05 (Table B01003).

**Bounds:** The share is clamped to `[0.0001, 0.999]` to avoid numerical instability at
the extremes (very tiny places or a municipality that covers almost the entire county).

### 2.2 Growth-Adjusted Share

When a municipal growth rate is available (from `growth-rates.json`):

```
relative_growth_rate = (1 + municipal_cagr) / (1 + county_cagr) - 1

adjusted_share(t) = pop_share_base × exp(relative_growth_rate × t)
```

Where `t` is years from the base year. This allows the municipal share to evolve over the
projection horizon as the municipality grows faster or slower than the county.

**When unavailable:** `adjusted_share = pop_share` (flat share; labelled *Estimated*).

### 2.3 Household Derivation

```
municipal_households = municipal_population × county_headship_rate
```

Where `county_headship_rate = county_households / county_population`.

If an ETL-derived municipal headship rate is available in `data/hna/derived/geo-derived.json`,
it takes precedence over the county rate.

## 3. Housing Stock Estimation

### 3.1 Total Units

```
total_units = municipal_households / (1 - county_vacancy_rate)
```

Where `county_vacancy_rate` comes from ACS 5-year DP04 (Table B25002).

If direct Census place-level housing unit data is available (`DP04_0001E`), it is used
instead and labelled *Direct*.

### 3.2 Tenure Split

```
renter_units = total_units × county_renter_rate
owner_units  = total_units - renter_units
```

Where `county_renter_rate = county_renter_occupied / county_total_occupied` (ACS DP04).

### 3.3 Structure Type Breakdown

County structure-type proportions (ACS DP04 fields `DP04_0014E` through `DP04_0021E`)
are applied to the municipal total:

```
municipal_sf_units        = total_units × county_sf_fraction
municipal_multifamily_units = total_units × county_mf_fraction
municipal_mobile_units    = total_units × county_mobile_fraction
```

For larger municipalities (pop ≥ 5,000) where place-level ACS structure data is available,
direct figures are substituted for the county proportions.

## 4. Affordability Tier Distribution

### 4.1 Baseline County Distribution

The county AMI tier distribution is derived from ACS 5-year DP04 GRAPI (Gross Rent as a
Percentage of Income) bins and DP03 income distribution data:

| Tier | AMI range | ACS GRAPI bin mapping |
|---|---|---|
| `tier_0_30` | 0–30% AMI | Renters with GRAPI ≥ 50% (severely burdened proxy) |
| `tier_30_60` | 30–60% AMI | Renters with GRAPI 30–49.9% (burdened proxy) |
| `tier_60_80` | 60–80% AMI | Renters with GRAPI 20–29.9% |
| `tier_80_120` | 80–120% AMI | Renters with GRAPI 10–19.9% |
| `tier_120_plus` | 120%+ AMI | Renters with GRAPI < 10% |

### 4.2 Municipal Rent Adjustment

```
rent_adj_factor = municipal_median_rent / county_median_rent
                  (clamped to [0.50, 2.00])
```

The adjustment is applied using a graduated weight vector:

```
If rent_adj_factor > 1 (more expensive):
  weights = [1.10, 1.05, 1.00, 0.95, 0.90]  (shift toward lower tiers)

If rent_adj_factor < 1 (cheaper):
  weights = [0.90, 0.95, 1.00, 1.05, 1.10]  (shift toward higher tiers)

adjusted_tier[i] = county_tier[i] × weights[i]
normalised_tier[i] = adjusted_tier[i] / sum(adjusted_tier)
```

### 4.3 Rent Burden Rate

```
municipal_rent_burden_rate = county_grapi_30plus × rent_adj_factor
                              (clamped to [0.01, 0.99])
```

### 4.4 Income Needed to Buy

Computed using standard mortgage underwriting:

```
loan_amount      = municipal_median_home_value × (1 - 0.10)   [10% down]
monthly_rate     = 0.07 / 12                                   [7% annual]
n_payments       = 360                                         [30-year fixed]
monthly_payment  = loan × (r × (1+r)^n) / ((1+r)^n - 1)
income_needed    = (monthly_payment / 0.30) × 12               [30% PTI rule]
```

Where `municipal_median_home_value` is taken from direct ACS place data if available,
otherwise from the county median.

## 5. Demographic Projections

### 5.1 Base Year

The base year is the most recent ACS 5-year vintage year (currently 2022, labelled 2024
in DOLA SDO forecasts which use 2024 as the projection anchor).

### 5.2 County Projection Components

DOLA SDO county components-of-change forecasts provide:
- Annual births and deaths by 5-year age cohort.
- Net migration (domestic + international) as an annual rate.

These are cached in `data/hna/projections/{county_fips}.json`.

### 5.3 Municipal Scaling

```
municipal_pop(t) = county_pop(t) × share(t)

share(t) = share_base × exp(relative_log_growth × t)

relative_log_growth = ln(1 + municipal_cagr_relative)
```

Where `municipal_cagr_relative = municipal_smoothed_rate - county_historical_rate`.

### 5.4 Household and Unit Derivation

```
households(t) = municipal_pop(t) × headship(t)

headship(t) = headship_base + headship_slope × t   ["trend" mode]
            = headship_base                         ["hold" mode]

units_needed(t) = households(t) / (1 - target_vacancy)
```

Default `target_vacancy = 0.08` (8%). This represents a healthy vacancy rate that allows
for normal market turnover.

## 6. Employment Estimation

### 6.1 LEHD WAC Scaling

```
municipal_jobs = county_total_jobs × municipal_pop_share
```

Wage tier breakdown:

```
municipal_low_wage_jobs  = county_CE01 × municipal_pop_share
municipal_mid_wage_jobs  = county_CE02 × municipal_pop_share
municipal_high_wage_jobs = county_CE03 × municipal_pop_share
```

**Important limitation:** LEHD WAC data is published at the county / workforce area level.
The population share scaling assumes uniform job density across the county, which is
generally not true. Urban centers typically have a higher job-to-population ratio than
suburban or rural areas. This limitation is reflected in the *Estimated* confidence label
applied to all municipal employment metrics.

### 6.2 Future Enhancement

Future versions of the ETL pipeline may ingest LEHD LODES RAC (Residence Area
Characteristics) and OD (Origin-Destination) tables at the block-group level, which
would allow more accurate place-level job counts and true commuting flow analysis.

## 7. Prop 123 Scaling

### 7.1 Eligibility

```
eligible = (municipal_population >= 1000)
```

Per HB 22-1093, counties must have ≥ 5,000 population. The municipality threshold is
≥ 1,000 population.

### 7.2 60% AMI Baseline

```
municipal_renters         = municipal_renter_units   (from housing stock estimate)
baseline_60ami_rentals    = municipal_renters × county_ami60_rental_fraction
```

Where `county_ami60_rental_fraction` is the fraction of county renter households
with incomes ≤ 60% AMI, derived from ACS DP03 income distribution and ACS DP04 tenure.

### 7.3 Growth Targets

```
target(n_years) = baseline × (1 + 0.03)^n_years
```

## 8. Confidence Scoring Rules

The `getDataConfidence(dataSource, municipalSize)` function applies the following rules:

| Input | Population threshold | Output confidence |
|---|---|---|
| `'direct'` | any | DIRECT (100) |
| `'interpolated'` | ≥ 2,500 | INTERPOLATED (80) |
| `'interpolated'` | < 2,500 | ESTIMATED (60) |
| `'estimated'` | any | ESTIMATED (60) |
| `'unavailable'` | any | UNAVAILABLE (0) |

## 9. Known Limitations and Caveats

1. **Vacation / resort communities:** ACS 5-year resident population may differ
   significantly from seasonal occupancy. The housing stock includes second homes
   and vacation rentals, which inflates the vacancy rate and skews affordability metrics.

2. **Rapidly growing municipalities:** The 5-year ACS vintage lags current conditions
   by 2–5 years. For fast-growing communities (CAGR > 3%), the population and household
   estimates may be materially understated.

3. **Annexations:** Municipal boundary changes between Census vintages can cause
   apparent population discontinuities. Always verify the geography definition when
   comparing across years.

4. **Income distribution:** The AMI tier redistribution algorithm uses a simple linear
   adjustment based on the rent differential. It does not account for the full joint
   distribution of income and rent, nor for changes in household composition.

5. **LEHD vintage lag:** LEHD LODES WAC data lags the current year by 12–18 months.
   Employment estimates may not reflect recent economic shocks or growth.

## 10. ETL Pipeline Integration

### Adding a New Municipality

1. Add an entry to `data/hna/municipal/municipal-config.json` with the correct
   7-digit Census place FIPS and 5-digit county FIPS (both zero-padded strings).
2. Add a corresponding entry to `data/hna/municipal/growth-rates.json`.
3. Run `python scripts/hna/build_hna_data.py --place <geoid>` to generate summary,
   LEHD, and projection cache files for the new place.
4. Run `python scripts/rebuild_manifest.py` to update the data manifest.

### Scaling Factor Cache

The `data/hna/municipal/scaling-factors/{fips5}.json` files are generated by the ETL
and contain pre-computed scaling inputs (population shares, headship rates, relative
growth rates) for every place within a county. These are consumed by the HNA JavaScript
to avoid re-computing shares on the client side.

Schema:

```json
{
  "countyFips5": "08077",
  "countyName": "Mesa County",
  "generated": "2024-11-15T00:00:00Z",
  "places": {
    "0831660": {
      "share0": 0.382,
      "relative_pop_cagr": 0.0089,
      "headship_base": 0.413,
      "headship_slope_per_year": -0.001
    }
  }
}
```
